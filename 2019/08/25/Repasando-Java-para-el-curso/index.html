  • length(): devuelve el número de caracteres por el que esta formado la string.
  • replace(char oldChar, char newChar): remplaza en una string un carácter por otro.
  • split(String regex): separa en string sustituyendo el valor indicado por un espacio en blanco.
  • substring(int beginIndex): devuelve otra string derivada de eliminar el numero de caracteres indicados de izquierda a derecha.
  • valueOf(boolean b): convierte diferentes tipos de valores en una string.

Reglas de estilo

Las reglas de estilo en Java son bien sencillas:

  • Variables y Métodos: empiezan por letra minúscula, si el argumento es algo del estilo first name, la primera palabra en minúscula y la segunda en mayúcula. Ejemplo: firstName.
  • Clases y Interfaces: La primera letra en mayúscula, si la clase tiene dos palabras la segunda también empieza por Mayúscula.
  • Paquetes: en minúscula y lo suyo es procurar que sea solo una palabra.
  • Constantes: en mayuculas totalmente.Ejemplo: PI, IPMF.

Documentacion: Javadoc

Para saber que es lo que hace Javadoc solo hay que echarle un ojo a la documentación oficial aqui.

Todo ese sitio web, con sus estilos, links y demás es en sí un doc de Java y con Javadoc podemos crear un doc exactamente igual para documentar nuestros proyectos.

Para ello solo hay que comentar el código con el tipo de comentario que indica a Javadoc lo que queremos que aparezca junto a sus etiquetas o tags.

Algunas de ellas son:

  • @author : indica el autor.
  • @param : parámetros de entrada de un método.
  • @return : valores devueltos por un método.
  • @deprecated : indica si un metodo no va a ser nunca mas usado.
  • @see : sirve para crear una lista y se combina con @link, nunca lo he utilizado así que lo desconozco.
  • {@link} : links a otras otras clases o métodos del javadoc.
  • @since : indica la versión desde la ultima modificación.
  • @throws or @exception : sirve para indicar las excepciones generadas en un método.
  • @Override : indica si un método ha sido reescrito, útil si se hereda de clases abstractas o interfaces.

E/S básica en Java

En java la E/S se realiza mediante streams. Un stream (canal) es un flujo de datos, una conexión entre el programa y la fuente de destino.

Existe tres streams de datos:

  • Streams de entrada de datos: Standart Input mediante System.in

  • Streams de salida de datos: Standart Onput mediante System.out y System.err

En la salida de datos tanto System.out como System.err nos permiten hacer escrituras con formato a través de métodos print, printf, println, etc.

En la entrada de datos surge un problema, y es que System.in solo maneja bits, lo que haría la gestión de datos demasiado compleja. Como solución, se utiliza la clase Scanner que se encuentra en el paquete java.util.

Métodos principales de la clase Scanner:

  • nextDouble(): lee la siguiente sentencia como un double y lo devuelve.
  • nextLine(): avanza una linea, evitando problemas de buffer y devuelve lo leído como una string.
  • nextInt(): lee la siguiente sentencia como un entero y lo devuelve.

Modificadores de acceso

En Java existen modificadores de acceso, los cuales nos ayudan a encapsular mas o menos un método, una clase o una variable/atributo:

  • public: se suele aplicar a clases, métodos y constantes. Estos pueden ser accedidos desde cualquier ámbito.
  • private: se suele aplicar a atributos, es decir a las variables de nuestras clases o a algún que otro método que solo queramos usar internamente.
  • protected: permite el acceso a componentes de la misma clase, mismo paquete o clases que heredan de esta.
  • default: es lo mismo que no usar ningún modificador de acceso, las clases y los atributos pasarán a ser publicas por defecto.

Modificadores de uso y ámbito

El ámbito define los límites en los que existe una variable o un método.

Se dividen en:

  • De instancia: son aquellos elementos (variables o métodos) que pertenece a cada objeto (instancia de objetos). Es el comportamiento por defecto visto hasta ahora.
  • De clase: son aquellos elementos (variables o métodos) que son únicos para la clase y todas sus estancias (objetos instanciados).
  • Locales o argumentos: de ámbito local o locales al método donde son declarados.

Para que un método o variable que pertenezca solo a ese ámbito se declararía anteponiendo la palabra static.

La herencia

Para declarar una subclase utilizando la herencia en Java es tan sencillo como escribir extends seguido de la subclase:

Reglas:

  • Solo se puede heredar de una única clase de una forma directa (herencia simple).

  • Si una clase no hereda explícitamente de otra, por defecto heredara de la clase java.util.Object.

  • Una subclase hereda todos sus métodos y atributos de su superclase, y de todos sus ancestros, hasta llegar a Object.

  • Una subclase puede añadir nuevos elementos (Atributos/Métodos) y también puede modificar “tapar” elementos heredados.

Para modificar un método de la superclase o clase superior, se haría copiando la cabecera y modificando el código de este.

La herencia crea una jerarquía de clases de forma que los permisos se distribuyen de esta forma:

public Y Y Y Y

Polimorfismo

En una relación de tipo herencia, un objeto de la superclase puede almacenar un objeto de cualquiera de sus subclases.

Si tenemos tres clases de las cuales Vehiculo actúa como la superclase y las otras dos (Camión y Motocicleta) heredan de ella reescribiendo uno de sus métodos:

Un objeto puede comportarse de un modo u otro dependiendo de como este sea instanciado, normalmente esto se hace de abajo a arriba (upcasting).

Otro modo de aplicar polimorfismo es el contrario, de arriba a abajo (downcasting) pero se necesita hace un casting para compilar. Es decir, a la hora de ejecutar el programa para que no de error es necesario indicarle el tipo de referencia de dicho objeto.

Si en tiempo de ejecución se necesita determinar si un objeto es de un cierto tipo u otro, siempre se puede usar el operador instanceof. Lo cual puede ser necesario para hacer un correcto casting al tipo adecuado.

El polimorfismo influye en la forma en que se resuelve la llamada de un método en ejecución. El método que se ejecuta depende del tipo de objeto en ese momento.

Interfaces

Una interfaz es un conjunto de métodos abstractos (sin implementación) y valores constantes que definen un contrato que debe ser cumplido por cualquier clase que lo implemente.

Implícitamente las constantes son public, final y los métodos public, abstract.

Reglas:

  • Una interfaz proporciona un mecanismo para facilitar que su comportamiento (métodos) pueda ser implementado por distintas clases de forma . no dependiente a la herencia. La interfaz proporciona un mecanismo para facilitar que su comportamiento (métodos) pueda ser implementado por distintas clases de forma independiente a la herencia.

  • Una interfaz puede extender mas de una interfaz pero no se puede extender clases. Una interfaz hereda todas sus constantes y métodos a no ser que estos queden tapados o sobrescritos.

  • Una interfaz es un tipo de datos referencia, y como tal puede utilizarse, tanto en argumentos de métodos como en declaración de variables. Además se aplica el polimorfismo con cualquier clase que la implemente de forma similar a como se hace con la herencia.

Excepciones

Una excepción es una indicación de que se ha encontrado una situación anómala durante la ejecución en el programa. En java el tratamiento de errores se encuentra en una clase mas.

Las excepciones permiten ejecutar el programa y evitar que este se rompa en tiempo de ejecución.

En Java se pueden hacer dos cosas con excepciones:

  • Capturar excepciones con el bloque try-catch-finally que permite gestionar los errores producidos durante la ejecución en un conjunto de sentencias.
  • Lanzar excepciones:
    • Creando una excepción, un objeto que describa la anomalía en cuestión.
    • Indicando que se ha detectado la anomalía con la palabra throw.

Bloque try-catch-finally:

En un main, este bloque se suele escribir dentro de un do-while junto a un boolean como condición del while.

Las excepciones pueden ser detectadas por la JVM o lanzadas por el programador con throw.
